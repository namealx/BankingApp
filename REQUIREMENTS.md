# Спецификация требований BankingApp

**Версия:** 1.0  
**Статус:** Утверждён

---

## 1. Назначение приложения
BankingApp — мобильное банковское приложение для платформы iOS, предназначенное для предоставления клиентам удалённого доступа к финансовым услугам. Приложение обеспечивает управление банковскими счетами, выполнение переводов, просмотр актуальных курсов валют, поиск ближайших отделений банка, а также управление профилем пользователя. Целевая аудитория — физические лица, являющиеся клиентами банка.

---

## 2. Общее описание

### 2.1 Пользователи
- **Авторизованный клиент:** Имеет доступ к полному функционалу: счета, переводы, курсы валют, карта, профиль.
- **Гость (неавторизованный):** Имеет доступ только к экрану входа и просмотру публичных курсов валют.

### 2.2 Требования к программному обеспечению
- **Xcode:** 26.5 или выше
- **iOS SDK:** 26.5
- **Swift:** 6.3
- **SwiftUI:** iOS 26.5 SDK
- **SQLite.swift:** 0.15.3 или выше
- **macOS:** Tahoe 26.2 или выше

### 2.3 Требования к аппаратному обеспечению
- **Целевое устройство:** iPhone 17 Pro
- **Архитектура:** Apple Silicon (ARM64)

## 3. Спецификация требований

### 3.1 Функциональные требования (FR)
- **FR-1:** Аутентификация по логину и паролю (проверка по локальной БД SQLite).
- **FR-2:** Просмотр списка активных счетов (текущий, сберегательный, кредитный, карт-счёт) с балансом и валютой.
- **FR-3:** Создание нового счёта и закрытие существующего с переносом остатка средств.
- **FR-4:** Переводы между своими счетами с автоматической конвертацией по внутреннему курсу.
- **FR-5:** Отображение интерактивной карты (MapKit) с маркерами отделений банка и маршрутами.

### 3.2 Удобство использования (UX/UI)
- Соответствие Apple Human Interface Guidelines (HIG).
- Поддержка Dynamic Type (масштабирование шрифтов).
- Нативная поддержка Светлой и Тёмной тем оформления интерфейса.

### 3.3 Требования к надёжности и безопасности
- Хранение паролей в виде SHA-256 хеша с уникальной солью для каждого пользователя.
- Автоматическое закрытие сессии при переходе приложения в фоновый режим.
- Использование транзакций SQLite для обеспечения атомарности переводов (ACID).

### 3.4 Требования к производительности
- Время холодного старта приложения: не более 2.0 секунд.
- Время выполнения локального SQL-запроса: не более 0.1 секунды.
- Плавность анимаций интерфейса: стабильные 60/120 FPS (ProMotion).

## 4. Сценарии использования (Use Cases)

### UC-1: Межсчётовый перевод средств
1. Пользователь выбирает счёт-источник.
2. Пользователь выбирает счёт-получатель в другой валюте.
3. Вводит сумму перевода.
4. Система отображает сумму списания и итоговую сумму зачисления по текущему курсу.
5. Пользователь подтверждает перевод.
6. В транзакции SQLite происходит списание, зачисление и логирование в историю.
